# QuickCard for AnkiDroid

QuickCard for AnkiDroid is an Android utility application designed to streamline the process of creating high-quality language flashcards for AnkiDroid. By providing offline Wiktionary data, it helps language learners build their own cards with minimal friction.

## 🚀 Features

- **Direct AnkiDroid Integration**: Automatically create cards in AnkiDroid using its API. No manual exporting or importing required.
- **Offline Wiktionary**: High-speed, offline searches using data derived from Kaikki.org.
- **Smart Card Creation**: Select definitions and example phrases directly from the search results to populate your cards.
- **Workflow Integration**:
    - **System-wide Search**: Highlight any word in any Android app and use the "Process Text" menu to instantly search it.
    - **Enqueue System**: Don't want to stop reading? Enqueue words to process them later.

## 📖 Usage

For detailed instructions on how to set up and use the app, please refer to the [Usage Guide](docs/usage/USAGE.md).

## 🛠 Project Structure

The project consists of two main modules:

1.  **`:app`**: The Android application containing the UI, dictionary integrations, and AnkiDroid bridge.
2.  **`:processor`**: A standalone Java utility used to convert large JSONL dumps from [Kaikki.org](https://kaikki.org) into optimized SQLite databases for the mobile app.

## 📋 Prerequisites

- **AnkiDroid**: You must have [AnkiDroid](https://play.google.com/store/apps/details?id=com.ichi2.anki) installed on your device.
- **API Access**: In AnkiDroid, go to `Settings > AnkiDroid > 3rd party API access` and ensure it is enabled.

## ⚙️ Setup

### Firebase Configuration
This app uses Firebase for several features:
- **Remote Config**: For dictionary discovery and mirror management.
- **Firestore**: For the feedback system.
- **Analytics/Crashlytics**: For quality monitoring.

To build the app yourself:
1. Create a Firebase project.
2. Register an Android app with the package name `com.github.lauroschuck.quickcard4ankidroid`.
3. Download the `google-services.json` and place it in the `app/` directory.

### Dictionary Hosting
The app discovers dictionaries via a JSON property in Remote Config. This JSON includes mirror URLs and metadata for the `.db` files generated by the `:processor`.

## 🏗 Building the Dictionaries

If you wish to generate your own offline databases:
1. Navigate to the root directory.
2. Run the processing script:
   ```bash
   ./process_dumps.sh
   ```
This script handles downloading the latest dumps, decompressing them, and running the `:processor` to generate the SQLite files and `metadata.json`.
